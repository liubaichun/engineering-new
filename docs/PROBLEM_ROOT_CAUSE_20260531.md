## 问题1：汉堡菜单按钮

### 现象
PC端桌面浏览器顶部左侧一直显示汉堡菜单按钮，无法点击。

### 根因分析
- CSS文件 `mobile-nav-toggle` 在桌面端应该 `display: none`
- 当前43和124服务器CSS均正常，第372行 `display: none` 存在
- 可能是浏览器缓存导致加载了旧CSS

### 修复方案
清除浏览器缓存 Ctrl+Shift+R 强制刷新

---

## 问题2：闪现左侧菜单栏

### 现象
客户管理、采购管理、运营管理、系统管理Tab切换时，左侧菜单栏闪现。

### 根因分析
CRM等模块使用iframe嵌套子页面，切换Tab时iframe内容短暂空白，父页面侧边栏从空白处透出来。

### 历史修复记录（2026-05-28）
已在CHANGELOG.md中记录：系统管理Tab跳转登录页修复

## 问题3：权限矩阵404

### 现象
点击系统管理 → 权限矩阵，显示"加载失败 400"。

### 根因分析

**代码历史**：
1. 2026-05-22 commit `faba18c` 删除了 `permission_registry` 系统
2. 删除了 `permission-matrix` API（`/api/core/user-company-permissions/matrix/`）
3. 模板页面 `permission_matrix.html` 仍调用已删除的API

**证据**：
```
权限矩阵页面JS调用：
fetch(`/api/core/user-company-permissions/matrix/?user_id=${state.userId}`)

API实际已删除，返回404
```

**当前权限系统**：
- 使用 `UserModulePermission (UMP)` 表
- `PermissionViewSet` 是只读的
- 没有提供权限矩阵编辑API

### 修复方案

**选项A**：禁用权限矩阵入口（最简单的方案）
**选项B**：重写权限矩阵页面，使用新的权限API
**选项C**：恢复旧API（不推荐，因为底层表已改变）

## 教训总结

1. **删除功能时要检查所有引用**
   - 删除API时要检查前端页面是否还在调用

2. **修改前要确认范围**
   - 不应该在未确认的情况下创建MenuGenerator

3. **验证流程**
   - 修改后必须验证
   - 不能直接同步到124服务器

4. **文档记录**
   - 所有修改都要记录根因
   - 避免重复犯错

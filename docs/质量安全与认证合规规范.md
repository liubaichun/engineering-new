### 1.2 三权分立

等保要求系统管理员、安全审计员、普通用户三权分离：

| 角色 | 职责 | 可操作 |
|------|------|--------|
| **系统管理员**（平台超管） | 租户管理、系统配置 | 创建/停用租户、配置全局参数 |
| **安全审计员**（独立角色） | 审计日志查看、安全事件处理 | 只能查看审计日志，不能操作业务数据 |
| **普通用户**（租户内用户） | 业务操作 | 只能操作自己权限范围内的业务 |

> 三权必须在数据库层面分离——审计员即使有管理员账号也无法修改审计日志。

### 2.1 密码策略

```python
PASSWORD_POLICY = {
    'min_length': 8,           # 最小长度
    'require_uppercase': 1,    # 至少1个大写字母
    'require_lowercase': 1,    # 至少1个小写字母
    'require_digit': 1,        # 至少1个数字
    'require_special': 1,      # 至少1个特殊字符
    'max_age_days': 90,        # 密码有效期90天（等保要求）
    'history_count': 5,        # 不能与前5次密码相同
    'max_attempts': 5,          # 连续失败5次锁定
    'lockout_minutes': 30,     # 锁定30分钟
}
```

**强制规则：**
- 初始密码必须要求用户首次登录后修改（已有 `password_changed` 字段）
- 密码传输必须使用HTTPS（TLS 1.2+）
- 密码存储必须使用bcrypt/argon2（禁止MD5/SHA1）
- 重置密码链接有效期15分钟

### 2.2 登录失败处理

```python
class LoginAttempt(models.Model):
    """登录尝试记录（等保审计要求）"""
    username = models.CharField('用户名', max_length=150)
    ip_address = models.GenericIPAddressField('IP地址')
    user_agent = models.TextField('客户端信息', blank=True)
    is_success = models.BooleanField('是否成功')
    fail_reason = models.CharField('失败原因', max_length=100, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        db_table = 'core_login_attempt'
        indexes = [
            models.Index(fields=['username', '-created_at']),
            models.Index(fields=['ip_address', '-created_at']),
        ]
```

**响应策略：**
```yaml
失败1-3次: 正常提示"用户名或密码错误"
失败4次: 提示"剩余1次尝试机会"
失败5次: 账号锁定30分钟 + 发送通知给管理员邮箱
锁定期间: 任何登录尝试都返回"账号已锁定，请30分钟后重试"
解锁方式: 等待30分钟自动解锁 / 管理员手动解锁
```


**审计日志保护措施：**
```yaml
# 等保二级强制要求
protection:
  - 日志表设置为 INSERT ONLY（禁止UPDATE和DELETE）
  - 日志独立存储（建议写入单独的日志数据库或日志平台）
  - 日志保留不少于6个月
  - 日志定期归档（6个月以上的压缩归档存储，保留不少于1年）
  - 日志不能被应用程序删除（只有平台超管通过数据库操作可清理）
  - 每天对日志完整性进行校验（逐条校验哈希值）
  - 日志时间不可篡改（使用NTP同步时间）
```

### 2.7 输入校验与防范（等保入侵防范要求）

```python
# Django REST Framework 全局异常处理
class GlobalExceptionHandler:
    """全局异常处理——等保要求不能泄露敏感信息"""

    def handle_validation_error(self, exc):
        # 返回通用错误信息，不暴露具体字段校验规则
        return Response({
            'code': 2001,
            'message': '输入参数校验失败',
            # ❌ 不返回具体哪个字段错了（防攻击者探测）
        }, status=400)

    def handle_authentication_error(self, exc):
        # 统一返回"用户名或密码错误"，不区分是用户不存在还是密码不对
        return Response({
            'code': 1001,
            'message': '用户名或密码错误',  # ✅ 统一提示
        }, status=401)
```

**防御清单：**
```yaml
SQL注入防护:
  - 全员使用ORM（禁止raw SQL，特殊场景必须参数化查询）
  - ALLOWED_HOSTS 严格配置，防止Host头注入
  - 文件上传校验（MIME类型+大小+内容扫描）

XSS防护:
  - 模板默认自动转义（Django自带，不要关闭）
  - 所有用户输入在前端展示前必须转义
  - 富文本内容使用白名单过滤（bleach库）

CSRF防护:
  - Django CSRF中间件全局启用
  - API端点使用Token认证（不存在CSRF问题）
  - Cookie设置 SameSite=Lax

请求限流:
  - 登录接口：每分钟5次/IP
  - 注册接口：每小时3次/IP
  - API接口：租户级别（见SaaS文档限流章节）
```

### 2.11 剩余信息保护

```python
# 数据删除规范
class DataDeletionPolicy:
    """等保二级要求——数据删除必须彻底"""

    # 软删除（标记删除）
    SOFT_DELETE_RETENTION = 30  # 天

    # 硬删除（真正删除）
    HARD_DELETE_AFTER = 30  # 软删除后30天

    def hard_delete_tenant(self, tenant_id):
        """彻底删除租户所有数据"""
        # 1. 删除业务数据
        for model in TENANT_MODELS:
            model.objects.filter(tenant_id=tenant_id).delete()

        # 2. 删除附件文件
        shutil.rmtree(f'/data/tenants/{tenant_code}/')

        # 3. 清理日志中的个人信息
        SecurityAuditLog.objects.filter(
            tenant_id=tenant_id,
            detail__has_keys=['id_card', 'phone', 'bank_card']
        ).update(detail={
            'note': 'personal_info_cleaned_for_compliance',
            'original_fields_deleted': True
        })

        # 4. 清除缓存
        redis.delete_pattern(f'*:{tenant_id}:*')

        # 5. 记录删除审计
        SecurityAuditLog.log(
            event_type='data_delete',
            detail={'tenant_id': tenant_id, 'scope': 'full_data_deletion'}
        )
```

```yaml
# RESTful API 标准
format: /api/{version}/{module}/{resource}[/{id}][/{action}]

示例:

  POST   /api/v1/finance/incomes/          # 创建
  GET    /api/v1/finance/incomes/{id}/     # 详情
  PUT    /api/v1/finance/incomes/{id}/     # 全量更新
  PATCH  /api/v1/finance/incomes/{id}/     # 部分更新
  DELETE /api/v1/finance/incomes/{id}/     # 删除

规则:
  - URL路径全小写+下划线（snake_case）
  - 资源名使用复数（incomes, clients）
  - 嵌套资源: /api/v1/crm/clients/{id}/contracts/
  - 动作通过HTTP方法表示，不在URL中使用动词
  - 特殊情况用 action 后缀，如 /api/v1/finance/incomes/{id}/approve/
```

```yaml
错误码范围:
  0:          成功

  1000-1099:  认证授权

    1002: Token已过期            → 返回401，刷新Token

    1004: 权限不足               → 返回403

  1100-1199:  租户相关
    1101: 租户不存在
    1102: 租户已停用
    1103: 租户用量超限

  1200-1299:  数据隔离
    1201: 跨租户访问被阻止
    1202: 数据权限不足

  2000-2999:  业务逻辑
    2001: 输入参数校验失败
    2002: 资源不存在
    2003: 资源已存在（重复）
    2004: 状态不允许当前操作
    2005: 操作被依赖资源阻止

  5000:       服务器内部错误（统一返回，不暴露细节）
```

```yaml
foreign_key_rules:
  必填字段（业务核心）:
    on_delete: PROTECT  # 禁止级联删除，防止误操作
    null: false
    blank: false

  可选字段:
    on_delete: SET_NULL
    null: true
    blank: true

  引用本公司的字段:
    company = ForeignKey(Company, on_delete=CASCADE, related_name='%(class)ss')

  related_name:
    必须填写（Django要求）

  索引:
    所有ForeignKey自动建索引
    复合索引: company + 常用查询字段
```

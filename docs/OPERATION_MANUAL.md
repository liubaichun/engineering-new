## 二、权限体系详解

### 2.1 权限模型

系统采用 **RBAC（基于角色的访问控制）** 模型：

```
用户（User）
  ├─ 系统级角色（User.role）：admin / manager / finance / staff / technician / viewer
  │    └─ 通过 UserRole（系统级角色权限表）关联 Permission
  │
  └─ 公司级角色（UserCompanyRole）：
       ├─ admin（公司管理员）：可管理本公司所有数据
       └─ staff（普通员工）：仅查看本公司数据
            └─ 通过 UserRole（公司级角色权限表）关联 Permission
```

### 2.2 系统级角色（6种）

| 角色代码 | 角色名称 | 说明 |
|----------|----------|------|
| `admin` | 超级管理员 | 拥有系统所有权限，可管理所有公司数据 |
| `manager` | 经理 | 可查看/管理大部分业务数据，无系统配置权限 |
| `finance` | 财务 | 财务模块完整权限（收入/支出/发票/工资） |
| `staff` | 普通员工 | 基础查看权限 |
| `technician` | 技术人员 | 设备/物料模块权限 |
| `viewer` | 查看者 | 只读权限 |

> **注意**：`admin` 角色的权限由系统硬编码控制（`is_superuser=True`），在角色管理页面为超级管理员分配具体权限无实际效果。

### 2.3 公司级角色（2种）

| 角色代码 | 角色名称 | 说明 |
|----------|----------|------|
| `admin` | 公司管理员 | 可管理本公司所有数据（多公司场景下每个公司可设独立管理员） |
| `staff` | 普通员工 | 仅查看本公司数据 |

公司级角色通过 **注册审批通过后自动分配**（公司管理员）或手动在 `/admin/core/usercompanyrole/` 中调整。

### 2.4 权限代码规范

格式：`资源:操作`，例如 `expense.create`

| 资源 | 说明 |
|------|------|
| `system` | 系统配置（权限/角色/用户/设置） |
| `approval` | 审批管理 |
| `project` | 项目 |
| `task` | 任务 |
| `finance` / `income` / `expense` / `invoice` / `wage` | 财务相关 |
| `client` / `contract` / `supplier` | CRM 相关 |
| `equipment` / `material` | 设备/物料 |
| `file` | 文件管理 |

| 操作 | 说明 |
|------|------|
| `read` | 查看 |
| `create` | 新建 |
| `write` / `update` | 编辑 |
| `delete` | 删除 |
| `export` | 导出 |
| `import` | 导入 |
| `approve` | 审批 |
| `manage` | 管理（含所有操作） |

### 2.5 权限分类（10类）

| 分类 | 说明 | 示例权限 |
|------|------|----------|
| 系统 | 权限/角色/用户/系统设置 | `system:role:manage`, `system:user:manage` |
| 审批 | 审批流程 | `approval:read`, `approval:approve` |
| 项目 | 项目管理 | `project:read`, `project:create` |
| 任务 | 任务管理 | `task:read`, `task:create` |
| 财务 | 财务通用 | `finance:read`, `finance:export` |
| 客户 | 客户管理 | `client:read`, `client:create` |
| 合同 | 合同管理 | `contract:read`, `contract:create` |
| 文件 | 文件管理 | `file:read`, `file:upload` |
| 工资 | 工资管理 | `wage:read`, `wage:create` |
| 其他 | 不属于以上分类 | 跨模块综合权限 |

### 3.2 配置步骤

1. 进入 **系统管理 → 角色管理**（`/system/roles/`）
2. 点击 **新建角色**，填写名称和代码
3. 在权限分配区，按模块勾选所需权限（支持模块级全选）
4. 点击 **保存**

> **提示**：搜索框支持按权限名称、代码或描述搜索，方便快速定位。

### 3.3 给用户分配系统级角色

1. 进入 **系统管理 → 用户-角色管理**（`/system/user-roles/`）
2. 找到目标用户所在行
3. 点击 **分配角色** 按钮
4. 在弹窗中勾选目标角色（可多选）
5. 确认分配

> **注意**：用户-角色管理只分配**系统级角色**（admin/manager/finance等）。公司级角色（公司管理员/普通员工）在注册审批时自动分配，无需手动处理。

## 四、注册审批流程（多租户闭环）

### 4.1 流程说明

```
新用户提交注册
    ↓
用户账号创建（is_active=False，等待审批）
    ↓
管理员在 Django Admin 审批（Approve）
    ↓
系统自动执行以下操作：
  ① 创建公司（Company），公司名=用户名
  ② 分配公司管理员角色（UserCompanyRole, role=admin）
  ③ 记录权限审计日志（PermissionAuditLog）
  ④ 发送站内通知
    ↓
用户收到通知，登录系统
    ↓
用户自动进入自己公司后台，只能看到本公司数据
```

### 4.2 审批入口

- Django Admin：`http://124.222.227.28:8001/admin/core/user/`
- 找到状态为 "待审批" 的用户，点击用户名进入详情页
- 点击页面底部 **其他操作** 菜单 → **审批通过** 或 **拒绝**

### 4.3 批量审批

在用户列表页，勾选多个待审批用户 → **其他操作** → **批量审批通过**

### 4.4 审批拒绝 vs 删除

- **拒绝**：账号保持 `is_active=False`，用户可修改信息后重新提交
- **删除**：彻底删除账号数据（不可恢复）

## 五、数据权限隔离（多租户Standalone模式）

### 5.1 数据可见范围规则

| 用户类型 | 数据可见范围 |
|----------|-------------|
| 系统超级管理员（admin） | 所有公司所有数据 |
| 公司管理员（company_admin） | 本公司所有数据 |
| 普通员工（staff） | 本公司数据（受权限控制） |

### 5.2 多公司数据隔离原理

所有业务数据模型（收入/支出/发票/项目等）包含 `company` 字段。

DRF ViewSet 通过 `get_queryset()` 方法注入 `company` 过滤：

```python
def get_queryset(self):
    qs = super().get_queryset()
    if request.user.is_superuser:
        return qs  # 超级管理员看全部
    # 自动按用户所属公司过滤
    return qs.filter(company=request.user.company)
```

### 5.3 切换公司（多公司任职用户）

如果一个用户同时属于多个公司，可在个人资料页切换当前公司，刷新后数据范围随之变化。

## 六、日常操作指南

### 6.1 用户注册与审批

| 操作 | 步骤 |
|------|------|
| 用户注册 | 访问 `/register/`，填写用户名/邮箱/密码/公司名 |
| 管理员审批 | Django Admin → core → Users → 点击用户名 → 审批通过 |
| 批量审批 | Django Admin → core → Users → 勾选用户 → 批量审批 |

### 6.2 权限配置（管理员）

| 操作 | 步骤 |
|------|------|
| 新建权限 | 系统管理 → 权限管理 → 新建权限（填写资源/操作/分类） |
| 新建角色 | 系统管理 → 角色管理 → 新建角色 → 勾选权限 → 保存 |
| 分配角色 | 系统管理 → 用户-角色管理 → 分配角色 |
| 查看权限被哪些角色使用 | 权限管理 → 点击某权限的编辑 → 无直接入口，可查审计日志 |

### 6.3 数据导入（Excel批量导入）

支持批量导入的模块：

| 模块 | 路径 | 说明 |
|------|------|------|
| 收入 | `/finance/income/import/` | Excel导入收入记录 |
| 支出 | `/finance/expense/import/` | Excel导入支出记录 |
| 发票 | `/finance/invoice/import/` | Excel导入发票 |
| 员工 | `/finance/employee/import/` | Excel导入员工 |
| 客户 | `/crm/client/import/` | Excel导入客户 |
| 供应商 | `/crm/supplier/import/` | Excel导入供应商 |
| 合同 | `/crm/contract/import/` | Excel导入合同 |

**导入格式要求**：
- 文件格式：`.xlsx`（Excel 2007+）
- 表头行必须与导出格式一致
- 日期格式：`YYYY-MM-DD`
- 金额：纯数字，不含货币符号

### 6.4 数据导出

各模块列表页均有 **导出** 按钮，支持 Excel 格式。审计日志也支持导出（`/audit/` 页面）。

### 6.5 审批流配置

系统根据金额自动触发审批：

| 模块 | 触发条件 | 审批人 |
|------|----------|--------|
| 支出（expense） | 金额 ≥ 1000 | 自动推送给财务角色 |
| 收入（income） | 金额 ≥ 5000 | 自动推送给财务角色 |

## 七、审计日志说明

### 7.1 记录范围

系统通过 Django 信号自动记录以下操作：

- **登录/登出**：时间、IP、用户名、结果（成功/失败）
- **数据增删改**：操作人、操作时间、数据类型、具体记录ID、变更内容
- **权限变更**：谁在什么时候给哪个角色增加了/删除了哪个权限
- **审批操作**：审批人、被审批对象、审批结果

### 7.2 查看审计日志

- 审计日志页面：`/audit/`
- 包含筛选器：按日期范围、操作类型、操作用户筛选

### 7.3 登录失败锁定

系统记录每次登录失败（`failed_login_attempts`），但**暂未接入自动锁定逻辑**（可手动在 Admin 中将用户 `is_active=False` 实现锁定）。

## 九、故障排查

### 9.1 用户无法登录

1. 检查账号是否已审批（`is_active=True`）
2. 检查浏览器 Cookie 是否开启
3. 清除浏览器缓存后重试

### 9.2 看不到本公司数据

1. 确认本公司已被创建（注册审批通过后自动创建）
2. 确认本公司 `is_active=True`（公司管理页面检查）
3. 切换当前公司（个人资料页）

### 9.3 权限不足（403 Forbidden）

1. 确认当前账号有所需的**系统级角色**（不是公司级角色）
2. 确认该角色包含所需的**权限码**
3. 联系管理员在角色管理中添加对应权限

### 9.4 审批流未触发

检查金额是否达到触发阈值（支出≥1000，收入≥5000）。

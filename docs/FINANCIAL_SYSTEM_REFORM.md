# 财务系统改造方案

> 基于 2026-05-27 全量数据分析，参照《小企业会计准则》和行业最佳实践
> 目标：让企业主不用专业知识也能一眼看懂财务数据

---

## 一、当前系统问题诊断

### 1.1 科目体系不全

当前科目/分类在收入端4个、支出端5个，缺关键会计科目：

| 应有科目 | 缺失影响 |
|---------|----------|
| 预付账款 | 恒鑫兴50万设备定金无法正确挂账 |
| 短期借款 | 如需记录银行借款 |
| 其他应收款-个人 | 刘柏春借款、员工备用金无法明细跟踪 |
| 其他应付款-个人/公司 | 关联方往来债务方无对应科目 |
| 财务费用 | 银行手续费/利息支出混入"other" |
| 销售费用 | 市场/销售支出无法独立统计 |
| 其他收益 | 政府补贴（稳岗补贴）无对应分类 |

### 1.2 收入分类错误（已发现7类，金额重大）

|:-:|---------|------|:----:|:--------:|:------:|
| 1 | 投资款当收入 | 普斯盾→绿聚能5笔 | 93.5万 | 主营业务收入 | 实收资本 ✅ 已修正 |
| 2 | 退款当收入 | 恒鑫兴3笔 | 50万 | 主营业务收入 | 内部往来（冲预付）✅ 已修正 |

| 4 | 往来款被误标 | 捷艺达2笔(ID 1012,1019) | 11万 | 主营业务收入 | 内部往来 ✅ 已修正 |

### 1.3 支出分类问题——"other"是垃圾桶

当前支出有640条标为`other`（占支出总条数640/975=65.6%），实际包含：

| 实际类型 | 条数 | 典型金额 | 建议新分类 |
|---------|:---:|:--------:|:----------:|
| 银行手续费 | 26 | 1.76~150元 | **财务费用** |
| 采购/货款/材料 | 83 | 1,000~560,080元 | **主营业务成本** |
| 技术服务费/服务费 | 若干 | 2,000~370,000元 | **主营业务成本**（有发票）或**管理费用** |
| 往来/借款/备用金 | 多条 | 大额 | **内部往来**（从支出表独立） |
| 离职补偿金 | 2 | 25万+44.7万 | **管理费用** |
| 社保/公积金代收代付 | 数条 | 252元/月 | **代收代付**（不属费用） |

### 1.4 关联方往来无跟踪

internal_transfer只贴标签，没有台账（已通过RelatedPartyLedger模型解决 ✅）

### 1.5 员工数据缺失导致工资记录不全

- 李斌、邓红艳、赖利在金易豪的Employee表不存在 → WageRecord无法录入
- Expense已有银行流水（半年薪+年终奖），但系统不认
- 买了社保是正式员工，应该补充Employee后自动按月计提

## 二、收入分类体系重构

### 2.1 当前收入分类状态（7个，5个真实+2个排除）

| 分类 | 内部代码 | 条数 | 金额(¥) | 说明 |
|:----|:--------:|:---:|:--------:|:----|
| 主营业务收入 | main_business | 95 | 15,305,149 | 核心业务（服务费、销售等） |
| 内部往来 | internal_transfer | 63 | 5,245,028 | 关联方资金往来（非收入）✅ 含生育津贴27,095 |
| 营业外收入 | non_operating | 53 | 23,097 | 小额杂项收入 |
| 实收资本 | equity | 7 | 975,000 | 投资款（非收入）✅ 新增2笔(含金易豪李斌2万) |
| 其他收益 | other_income | 1 | 595 | 稳岗补贴 ✅

### 2.2 建议新增收入分类

| 新增分类 | 代码 | 适用场景 | 涉及数据 |
|:---------|:----:|:---------|:--------:|
| **其他业务收入** | other_business | 租金、材料销售、技术服务费（非主业） | 外租车租金4万、材料销售等 |
| **其他收益** | other_income | 政府补助、稳岗补贴、个税手续费返还 | 百川稳岗补贴594 |
| **投资收益** | investment_income | 理财收益、投资分红 | 暂缺数据，结构预留 |

### 2.3 最终收入分类体系（建议7个）

```
收入分类（5个真实 + 2个非收入排除）：

  真实收入（入利润表）：
  ├─ 主营业务收入    main_business
  ├─ 其他业务收入    other_business      ← 新增
  ├─ 营业外收入      non_operating
  ├─ 其他收益        other_income        ← 新增
  └─ 投资收益        investment_income   ← 新增（结构预留）

  非收入（在收入表展示，但不入利润表）：
  ├─ 内部往来        internal_transfer
  └─ 实收资本        equity
```

## 三、支出分类体系重构

### 3.2 当前支出分类状态（重构后，9个）

| 分类 | 代码 | 条数 | 金额(¥) | 说明 |
|:----|:----:|:---:|:--------:|:-----|
| 税费 | tax | 299 | 1,327,065 | 含代理国库税收收缴30条（从other提出） |
| **主营业务成本** | main_cost | **253** | **5,921,464** | **新增172条：161条服务费+11条供应商支出** |
| 工资薪酬 | salary | 115 | 3,511,328 | 含年终奖7条+工资51条（从other提出） |
| 管理费用 | admin_expense | 93 | 2,730,036 | 费用报销/离职补偿/房费（从other提出） |
| 代收代付 | agency | 85 | 428,633 | 社保托收（从other提出） |
| 内部往来 | internal_transfer | 66 | 5,952,771 | 往来/借款/备用金（从other提出） |
| **财务费用** | finance_expense | **53** | **1,403** | **从214条清理到53条（移出161条服务费）** |
| 其他 | other | **10** | **629,498** | **从21条进一步清理到10条（需发票核实）** |
| 差旅费 | travel | 1 | 5,904 | 未变 |

> **本轮改造成果：** other从640条降至10条（-98.4%）；财务费用从214条降至53条（-75.2%）；主营业务成本从81条增至253条（+212%）

### 3.2 建议支出分类体系

**拆除"other"垃圾桶，按会计准则拆分：**

| 分类 | 代码 | 说明 |
|:----|:----:|:-----|
| **主营业务成本** | main_cost | 采购、货款、技术服务费、材料（有发票/合同） |
| **管理费用** | admin_expense | 办公费（已有）、离职补偿、行政支出 |
| **财务费用** | finance_expense | 银行手续费、利息支出/收入（红字） |
| **销售费用** | sales_expense | 招待费、广告、市场推广 |
| **工资** | salary | 员工工资（已有） |
| **税费** | tax | 税费（已有） |
| **内部往来** | internal_transfer | 关联方资金往来（非支出） |
| **代收代付** | agency | 社保代收代付、代扣代缴（不影响损益） |
| **其他** | other | 兜底（应大幅减少至5%以内） |

## 六、系统代码修改点

修改涉及以下文件，每条修改需更新Serializer+筛选+报表：

```
apps/finance/models.py
  └─ Income.income_category CHOICES  ← 新增3个分类
  └─ Expense.expense_type CHOICES    ← 拆分other为多个

apps/finance/serializers.py
  └─ IncomeSerializer/ExpenseSerializer ← 更新choice展示

apps/finance/classify_incomes.py
  └─ 分类规则更新（生育津贴、稳岗补贴）

apps/finance/reports_v2.py
  └─ 报表API更新（新分类统计、筛选条件）

apps/finance/admin.py
  └─ 如果新增字段需要管理员显示

templates/finance/
  └─ 收入管理/支出管理页面 ← 筛选下拉菜单更新
```

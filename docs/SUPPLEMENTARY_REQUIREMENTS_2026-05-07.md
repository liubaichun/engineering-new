## 四、技术债务（本轮发现）

| # | 问题 | 位置 | 修复方案 |
|---|------|------|----------|
| T-1 | `safeJson` 只处理 `data.results`，裸 list 数据被忽略 | `templates/stats.html` | 改为 `data.results || data.data || (Array.isArray(data) ? data : [])` |
| T-2 | gunicorn 多实例启动（历史进程残留 + 本次新开） | systemd/service | 确认用 systemd 管理，或杀进程脚本加 PID 文件检查 |
| T-3 | User 模型无 `company_id`，无法按公司过滤用户 | `apps/core/models.py` | User 加 `company_id`，影响：login/permission/user_admin |
| T-4 | 两套审批并存（Admin 审批 vs DRF ApprovalFlow） | 全系统 | 统一到 ApprovalFlow，废弃 AdminApproval 相关代码 |
| T-5 | 个税计算使用月减 5000，未用累计预扣法 | `WageRecord.calculate_gross_and_tax()` | 重构为累计预扣法 |
| T-6 | 社保/公积金为手工录入，未自动化 | `Employee` + `WageRecord` | 根据 `social_insurance_base` + 固定比例自动计算 |

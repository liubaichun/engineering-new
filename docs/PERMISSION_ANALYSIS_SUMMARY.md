# 权限管理系统完整分析报告

## 一、系统架构全景

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户请求                                 │
└────────────────────────────┬──────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│              两套权限系统并行工作                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【系统1】API数据过滤                                            │
│  工具函数: get_user_companies()                                  │
│  数据来源: UserCompanyPermission (UCP)                            │
│  作用: 过滤queryset，只返回用户有权限的公司数据                     │
│                                                                 │
│  【系统2】权限检查                                                │
│  权限类: RoleRequired                                             │
│  数据来源: UserModulePermission (UMP) + Permission (权限码)       │
│  作用: 检查用户是否有权限执行某个操作                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、两套数据表的用途

| 表 | 用途 | 使用场景 |
|---|------|----------|
| `UserCompanyPermission` (UCP) | API数据过滤 | IncomeViewSet.get_queryset() |
| `UserModulePermission` (UMP) | 位掩码权限 | RoleRequired.has_permission() |
| `Permission` | 权限码定义 | 权限检查时查询权限码是否存在 |

## 三、发现的问题

### 问题1：Permission表不完整（129个缺失）

**现象**：
- Module表: 34条
- ModuleAction表: 136条
- Permission表: 299条
- 实际缺失: 129个权限码

**影响**：
- 权限检查时找不到对应的Permission记录
- 部分权限码检查会失败

### 问题2：UMP和UCR不一致

**yangxiaohui用户**：
- UCR公司: {4} (UserCompanyRole)
- UMP公司: {3, 4} (UserModulePermission)

**liubc用户**：
- UCR公司: {2, 3, 4, 6} (全部4家公司)
- UMP公司: {2, 3, 4, 6} 但分散在10个模块，每个模块只能访问不同公司

### 问题3：两套系统不一致导致的问题

| 检查项 | UMP系统 | UCP系统 |
|--------|---------|---------|
| 数据过滤 | 基于公司ID | 基于公司ID |
| 权限粒度 | 模块+动作位掩码 | 模块+动作单条记录 |
| 超管处理 | bypass | bypass |

**关键问题**：
- UCP表有5400条记录（liubc）
- UMP表只有10条记录（liubc）
- API数据过滤用UCP，权限检查用UMP，两套不一致

## 四、根本原因

1. **Permission表同步机制缺失**：ModuleAction注册时没有自动创建Permission记录
2. **UMP和UCR写入逻辑不同步**：分配角色时只写了一个表
3. **两套系统并行使用**：没有统一的数据流设计

## 五、修复方案

### 方案A：统一使用UMP系统（推荐）

```
1. 修复Permission表：补充129个缺失的权限码
2. 统一数据过滤逻辑：API层也使用UMP表
3. 修复UMP/UCR一致性：删除孤立UMP记录
```

### 方案B：保持现状但修复缺陷

```
1. 修复Permission表
2. 保持两套系统并行
3. 确保UMP和UCR同步更新
```

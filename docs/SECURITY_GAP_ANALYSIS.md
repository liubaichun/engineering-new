# 安全问题 & 商业差距分析
**日期:** 2026-04-28
**状态:** 待处理

---

## 高危（影响生产安全）

| ID | 问题 | 说明 | 优先级 |
|----|------|------|--------|
| S1 | **CSRF全局禁用** | `CSRFExemptSessionAuthentication` 让整个API放弃CSRF保护 | 🔴 P0 |
| S2 | **登录无频率限制** | 没有任何防暴力破解机制，可对任意账号无限次尝试密码 | 🔴 P0 |
| S3 | **密码哈希未强化** | 使用Django默认PBKDF2，如数据库泄露可跑字典猜测弱密码 | 🔴 P1 |

---

## 中危（生产环境必备）

| ID | 问题 | 说明 | 优先级 |
|----|------|------|--------|
| S4 | **无审计日志** | 密码修改、权限变更、审批通过等敏感操作无记录 | 🟡 P2 |
| S5 | **无安全响应头** | 缺少 CSP/HSTS/X-Frame-Options/X-Content-Type-Options | 🟡 P2 |
| S6 | **无文件扫描** | 上传文件无病毒扫描，Excel导入无宏安全检查 | 🟡 P2 |
| S7 | **无数据备份机制** | PostgreSQL无自动备份，数据泄露/误删无恢复手段 | 🟡 P2 |
| S8 | **敏感字段无加密** | 工资、社保等字段在数据库中明文存储 | 🟡 P2 |

---

## 功能性差距（商业系统标配）

| ID | 问题 | 说明 | 优先级 |
|----|------|------|--------|
| F1 | **无两步验证/OTP** | 账号泄露即全系统沦陷 | 🟠 P3 |
| F2 | **无SSO/OIDC** | 无法对接企业身份提供商（飞书/钉钉/企微） | 🟠 P3 |
| F3 | **无操作员行为审计** | 谁在什么时候改了什么，无完整日志 | 🟠 P3 |
| F4 | **审批节点未配置** | ApprovalTemplate.nodes全为空，审批系统不可用 | 🟠 P3 |
| F5 | **无数据导出脱敏** | Excel导出工资等敏感数据无脱敏处理 | 🟠 P3 |
| F6 | **无字段级权限** | 普通用户可能通过API看到所有员工薪资 | 🟠 P3 |
| F7 | **无消息通知中心** | 审批结果、任务变更等无站内消息推送 | 🟠 P3 |

企业信息化管理系统 GREENv2.0

2026-05-02 | /root/engineering-new

系统规模

  • • 9 个 Django App 模块
  • • 30+ 张数据库表 (PostgreSQL)
  • • 100+ REST API 端点
  • • 2 个代码分支 (master / standalone)
  • • 技术栈: Django 4.2 · DRF 3.14 · PostgreSQL · Gunicorn · Nginx
  • • 前端: Bootstrap 5 · jQuery · 原生 AJAX

部署checklist（每次必须执行）

  • • 1. git pull → 检查网络是否正常
  • • 2. python manage.py showmigrations --plan（查依赖链）
  • • 3. python manage.py check --deploy（安全检查）
  • • 4. python manage.py migrate（严格按顺序）
  • • 5. systemctl restart engineering-gunicorn
  • • 6. curl 冒烟测试核心 API（users/contracts/projects/）
  • • 7. 新服务器首次部署必须加 --fake-initial

已知部署坑点（禁止再犯）

  • • ❌ 跳过 migrate 错误用 --fake → 迁移图有"洞"永远补不上
  • • ❌ DB schema 和代码不一致 → inconsistent state
  • • ❌ 多租户代码部署到独立版 → 500错误（company_id 不存在）
  • • ❌ cherry-pick 新功能时不包含 migrations → 新表不存在
  • • ❌ 124 服务器 DB 缺 equipment/material 表（未执行migrate）
  • • ❌ notifications 迁移链顺序: 0003 depends on 0002_initial（不存在）
  • • ✅ 新增 model → 立即 makemigrations + commit + push

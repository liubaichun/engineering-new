# 通知渠道使用说明

## 一、什么是通知渠道？

通知渠道是系统用来向您推送消息的"通道"。系统支持以下渠道：

| 渠道 | 用途 | 需要什么 |
|------|------|---------|
| **飞书** | 通过飞书机器人发消息给您 | 飞书开放平台自建应用的 App ID 和 App Secret |
| **企业微信** | 通过企业微信应用发消息给您 | 企业微信管理后台的企业ID、应用Secret、应用AgentID |
| **钉钉** | 通过钉钉应用发消息给您 | 钉钉开放平台的 AppKey、AppSecret、AgentID |
| **邮件** | 通过 SMTP 发送通知邮件 | SMTP 主机、端口、账号、密码（授权码） |

## 二、两种通知方式

### 1. 推送通知（发给你个人）
管理员配置渠道后 → 你绑定你的账号 → 系统就能发消息给你

**示例：**
- 合同快到期了 → 你在飞书收到提醒
- 审批单需要你处理 → 你在企业微信收到消息

### 2. 广播通知（发到群）
管理员配置渠道时如果填了群机器人Webhook地址，系统可以直接发消息到群

**示例：**
- 公司公告 → 发到飞书全员群
- 系统维护通知 → 发到企业微信大群

## 三、管理员操作：配置渠道

**入口：** 系统管理 → 通知渠道（或直接访问 /channels/）

### 第一步：添加渠道

1. 点击右上角「添加渠道」
2. 选择渠道类型（飞书/企业微信/钉钉）
3. 填写配置信息（每个字段下方都有灰色的文字提示，告诉你从哪里获取）
4. 选择用途（私信通知或群通知）
5. 点击「保存」

> 💡 **提示：** 每个配置项下方都有👉指引，告诉你去哪里找这个参数。

### 第二步：验证连接

1. 在渠道卡片上点击「···」→「验证连接」
2. 显示"✓ 凭证有效"说明配置正确

### 第三步：发送测试消息

1. 在渠道卡片上点击「···」→「发送测试」
2. 检查你的飞书/企微是否收到测试消息

---

## 四、用户操作：绑定账号

管理员配好渠道后，每个用户需要绑定自己的账号才能接收通知。

### 扫码绑定（飞书/企业微信）

1. 在个人设置页点击「通知渠道绑定」
2. 在渠道卡片上点击「绑定」
3. 用飞书/企业微信扫码授权
4. 绑定成功后会在"我的绑定"列表中出现

## 五、企业微信配置详细指引

企业微信的配置最容易出错，以下是完整的配置步骤：

### 1. 创建自建应用

1. 登录 [企业微信管理后台](https://work.weixin.qq.com/wework_admin/)
2. 左侧菜单 → 应用管理 → 自建 → 创建应用
3. 上传应用Logo，填写应用名称（如"审批通知"）
4. 选择可见范围（建议选"全部成员"）
5. 点击「创建应用」

### 2. 获取配置参数

| 参数 | 从哪里找 | 操作步骤 |
|------|---------|---------|
| **企业ID** | 我的企业 → 企业信息 | 页面最上方显示的"企业ID"（以 wx 开头的一串字符） |
| **应用Secret** | 应用管理 → 点刚创建的应用 | 进入应用详情页，点击 Secret 栏的「查看」按钮 |
| **应用AgentID** | 应用管理 → 点刚创建的应用 | 进入应用详情页，AgentId 在 Secret 的上方 |

### 3. 设置可调用应用

为使系统能向成员发送消息，还需设置"企业可信IP"：

1. 应用管理 → 点应用 → 企业可信IP
2. 添加服务器IP地址（43服务器的IP：`43.156.139.37`）
3. 点击「保存」

### 4. 在系统中配置

1. 系统管理 → 通知渠道 → 添加渠道 → 选择"企业微信"
2. 依次填入上面的三个参数（企业ID、应用Secret、应用AgentID）
3. 点击「保存」
4. 点击「验证连接」确认配置正确

## 六、飞书配置详细指引

### 1. 创建自建应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 点击「创建应用」→「企业自建应用」
3. 填写应用名称（如"ERP通知机器人"）
4. 点击「确定创建」

### 2. 获取配置参数

| 参数 | 从哪里找 |
|------|---------|
| **App ID** | 应用详情 → 凭证与基础信息 → App ID |
| **App Secret** | 应用详情 → 凭证与基础信息 → App Secret（点击"显示"） |

### 3. 配置应用权限

1. 应用详情 → 权限管理 → 添加权限
2. 搜索并添加以下权限：
   - `im:message:send_as_bot`（以应用身份发送消息）
   - `contact:contact:readonly`（读取通讯录，用于获取用户信息）
3. 点击「批量申请」

### 4. 发布版本

1. 应用详情 → 版本管理与发布
2. 创建版本 → 填写版本号和更新说明
3. 提交审核 → 审核通过后点击「发布」

### 5. 设置事件回调（用于扫码绑定）

1. 应用详情 → 事件与回调
2. 回调配置：
   - 回调URL：`http://43.156.139.37:8001/api/channels/bind/callback/{channel_id}/`
   - 订阅事件：添加 `user.created`（用户首次绑定事件）

### 6. 配置群机器人（可选，用于群广播）

1. 打开飞书群 → 群设置 → 群机器人
2. 添加机器人 → 选择刚才创建的应用
3. 复制Webhook地址（以 `https://open.feishu.cn/open-apis/bot/v2/hook/` 开头）
4. 在系统渠道配置中填入此Webhook地址

### 7. 在系统中配置

1. 系统管理 → 通知渠道 → 添加渠道 → 选择"飞书"
2. 填入 App ID 和 App Secret
3. 如需要群广播功能，填入群机器人 Webhook 地址
4. 选择用途（私信通知或群通知）
5. 点击「保存」
6. 点击「验证连接」

## 七、常见问题

### Q: 验证连接失败，提示"凭证无效"

A: 最常见的原因是：
- 参数填写有误（仔细核对大小写和特殊字符）
- Secret 已重置（去对应平台重新查看并更新）
- 企业微信未设置可信IP（详见上面的第3步）
- 飞书应用未发布版本（必须发布后才能使用）

### Q: 用户绑定了但没有收到通知

A: 检查：
- 该用户是否在系统中有对应的系统账号
- 通知渠道是否已启用（状态显示"已连接"）
- 用「发送测试」确认渠道本身能工作

### Q: 如何给特定的人发通知？

A: 系统会根据事件自动判断接收人。例如：
- 审批任务 → 发给审批人
- 合同到期 → 发给合同负责人
- 任务完成 → 发给任务创建者

管理员不需要手动指定每个通知发给谁——系统会自动路由。

# 企业微信通知通道配置说明

## 前置条件

- 你已是企业微信管理员
- 服务器 IP 已固定（当前：`43.156.139.37`）

---

## 第一步：获取企业 ID（CorpID）

1. 打开 **企业微信管理后台** → https://work.weixin.qq.com/wework_admin/loginpage
2. 左下方 **「我的企业」** → **「企业信息」**
3. 复制 **「企业ID」**（如 `wxea2fd74d64fba451`）— 稍后需要填写到系统

---

## 第二步：创建自建应用

1. 左侧菜单 **「应用管理」** → 底部 **「自建」** → 点击 **「创建应用」**
2. 填写：
   - 应用名称：`ERP通知助手`（可自定义）
   - 应用图标：可上传
   - 应用简介：`接收系统通知消息`
3. 创建成功后进入应用详情页
4. 找到 **「AgentId」** 和 **「Secret」**：
   - **AgentId**：一个数字（如 `1000002`）
   - **Secret**：点击 **「查看」** → 复制该字符串（如 `5scVmuQ0GwOq0dQCoAeyl0R261nNrO4uF4RgkpKCuhE`）
5. ⚠️ **这三个值（CorpID、AgentID、Secret）是核心凭证，复制到文档备用**

---

## 第三步：设置应用可见范围

1. 在应用详情页找到 **「可见范围」**
2. 点击 **「设置可见范围」**
3. 选择要接收通知的人员或部门（至少要包含你自己）
4. 点击 **「确定」**

> 只有在此范围内的成员才能收到消息，不在范围内的人 API 返回成功但实际收不到。

---

## 第四步：配置接收消息服务器（回调 URL）

1. 在应用详情页找到 **「接收消息」** → 点击 **「设置接收消息」**（或 **「配置」**）
2. 填写以下参数：
   - **URL**：`http://43.156.139.37:8001/api/channels/wecom/verify/`
   - **Token**：`erpnotify2026`
   - **EncodingAESKey**：点击 **「随机获取」** 生成（43位字符）
3. 点击 **「保存」**，系统会验证 URL 有效性
4. 验证成功后，状态应显示 **「已启用」**

---

## 第五步：配置企业可信 IP

> ⚠️ 这一步最关键且容易搞错：**不是**配置「通讯录同步」的 IP 白名单，而是配置应用级别的 IP。

1. 在应用详情页，找到 **「企业可信IP」**（通常在"开发者接口"区域）
2. 点击 **「配置」**，添加 IP：`43.156.139.37`
3. 点击 **「保存」**
4. （可选也可同时添加备用 IP：`43.156.139.37`）

> 如果不配这一步，API 会返回错误码 **60020**（不安全的访问IP），消息发不出去。

---

## 第六步：获取你自己的 userid

1. 打开 **企业微信 App**
2. 进入 **「我」** → 点击头像进入个人信息页
3. 查看 **「帐号」**（即 userid，如 `liubc`）
4. 如果看不到，让管理员在管理后台 **「通讯录」** 中查看你的 **「帐号」**

---

## 第七步：在 ERP 系统中添加通知渠道

1. 登录 ERP 系统 → **「通知渠道」**
2. 点击 **「添加渠道」**
3. 填写：
   - 渠道名称：`企业微信通知`（可自定义）
   - 渠道类型：**企业微信**
   - 用途：**私信通知**
   - 企业 ID（CorpID）：填入第一步获取的值
   - 应用 AgentID：填入第二步获取的值
   - 应用 Secret：填入第二步获取的值
4. 点击 **「保存」**

---

## 第八步：绑定用户

1. 在通知渠道页面的「用户绑定」区域，点击 **「添加绑定」**
2. 选择：
   - 系统用户：**admin**（或你要绑定的用户）
   - 渠道：**企业微信通知**
   - 平台账号标识：填入你第六步获取的 **userid**（如 `liubc`）
3. 点击 **「保存」**

---

## 验证

1. 在渠道列表中找到「企业微信通知」，点击 **测试按钮（▶）**
2. 打开企业微信 App，在 **「消息」** 列表中应该收到一条测试消息
3. 如果收到，配置完成 ✅

---

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| API 返回 60020 | 未配置企业可信IP | 参考第五步 |
| API 返回 40001 | Secret 错误或已重置 | 重新查看 Secret 并在系统中更新 |
| 消息发送成功但收不到 | 不在可见范围内 | 检查第三步的可见范围 |
| 消息发送成功但收不到（2） | 未绑定用户 | 参考第八步 |
| 提示"您尚未绑定该渠道" | 私信通知需要绑定 | 参考第八步 |

---

## 一、产品定位与核心目标

### 1.1 产品定位

**服务器全息监控平台** —— 一键安装后自动适应服务器环境，自动收集所有运行数据，构建完整的服务器资产地图，并提供实时监控、告警通知和历史数据分析。

**核心特性：**
- 零配置：安装后自动运行，不需要人工干预
- 自适应：根据服务器环境自动识别应用类型和配置
- 可扩展：支持多台服务器统一监控
- 轻量化：Docker部署，不影响现有服务

### 1.2 终极目标

```
新服务器开机 → 安装平台 → 自动发现所有数据 → Web界面展示
```

**用户只需要做一件事：**
```bash
docker-compose up -d
```
然后打开浏览器访问 http://服务器IP:5000

### 1.3 一句话定义

> 摸清"跑了什么、放在哪、怎么连、稳不稳"。

## 二、系统架构

### 2.1 整体架构：混合方案C

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         【监控平台中心节点】                               │
│                                                                        │
│   Web界面(5000) ←→ 数据处理 ←→ 数据库(SQLite/PG) ←→ 定时任务            │
│                                              ↓                          │
│                                      告警引擎 → 飞书/企微通知            │
└─────────────────────────────────────────────────────────────────────────┘
           ↑                                    ↑                        ↑
    首次SSH安装Agent                    数据上报（实时）                   ↑
           │                                    │                        ↑
┌──────────┴─────────┐  ┌───────────────────────┴──────┐  ┌──────────────┴─────┐
│    Server A        │  │         Server B              │  │      Server C       │
│  首次：SSH安装Agent │  │    首次：SSH安装Agent         │  │   首次：SSH安装Agent │
│  之后：Agent自采集  │  │    之后：Agent自采集           │  │   之后：Agent自采集  │
│  端口: 9100        │  │    端口: 9100                  │  │   端口: 9100         │
└────────────────────┘  └──────────────────────────────┘  └────────────────────┘
```

### 2.2 工作流程详解

#### 阶段1：首次添加服务器（手动）
```
用户操作：在Web界面输入 Server B 的IP + SSH密码
        ↓
监控平台：SSH连接到 Server B
        ↓
监控平台：执行初始化脚本（安装Agent）
        ↓
Agent启动：在 Server B 本地运行，端口9100
        ↓
首次采集：Agent执行完整扫描，上报到中心节点
        ↓
完成：Server B 出现在Web界面的服务器列表中
```

#### 阶段2：日常运行（自动化）
```
Agent运行：每30秒自动采集最新数据
        ↓
Agent上报：增量数据上报到中心节点
        ↓
中心节点：存储到数据库，更新Web界面
        ↓
用户查看：打开浏览器实时查看 Server B 状态
```

#### 阶段3：告警触发（自动化）
```
Agent检测：CPU>85% 或 进程异常退出
        ↓
Agent上报：告警事件到中心节点
        ↓
中心节点：触发告警规则
        ↓
通知推送：飞书/企微消息发送给管理员
        ↓
用户处理：登录Web查看告警详情
```

### 2.3 两种部署模式

#### 模式1：单机独立模式
- 监控平台 + Agent 安装在同一台服务器
- 适合：个人使用、小规模测试
- 访问地址：http://本机IP:5000

#### 模式2：多服务器集群模式
- 中心节点：监控平台（选择一台常开服务器）
- 被监控服务器：安装轻量Agent
- 适合：团队使用、生产环境

### 2.4 安装方式

#### Docker一键部署（推荐）
```bash
# 单机模式
docker-compose up -d

# 访问
http://服务器IP:5000
```

#### 其他服务器安装Agent
```bash
# 在中心节点Web界面添加服务器IP和SSH密码后
# 系统自动SSH过去执行：
curl -fsSL http://中心节点:5000/static/install-agent.sh | bash
```

### 4.2 数据表设计

#### 4.2.1 服务器表 (servers)
```sql
CREATE TABLE servers (
    id INTEGER PRIMARY KEY,
    ip VARCHAR(45) UNIQUE NOT NULL,
    hostname VARCHAR(255),
    ssh_port INTEGER DEFAULT 22,
    ssh_user VARCHAR(64),
    group_id INTEGER,
    status VARCHAR(20),  -- online/offline/warning
    last_seen DATETIME,
    created_at DATETIME,
    updated_at DATETIME
);
```

#### 4.2.2 系统指标表 (system_metrics)
```sql
CREATE TABLE system_metrics (
```

## 六、部署方案

### 6.1 单机部署（Docker Compose）

```yaml
# docker-compose.yml
version: '3.8'

services:
  monitor:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - ./data:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - FLASK_ENV=production
      - SECRET_KEY=your-secret-key
    restart: unless-stopped

  agent:
    build:
      context: ./agent
      dockerfile: Dockerfile.agent
    network_mode: host
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - CENTER_NODE=http://localhost:5000
      - SERVER_ID=1
    restart: unless-stopped
```

### 6.2 Agent安装脚本（首次添加服务器时执行）

```bash
#!/bin/bash
# install-agent.sh
set -e

CENTER_NODE="$1"
SERVER_ID="$2"

# 下载Agent镜像或二进制
curl -fsSL ${CENTER_NODE}/static/agent-binary.tar.gz -o /tmp/agent.tar.gz

# 解压到指定目录
tar -xzf /tmp/agent.tar.gz -C /opt/

# 配置
cat > /opt/agent/config.env << EOF
CENTER_NODE=${CENTER_NODE}
SERVER_ID=${SERVER_ID}
COLLECT_INTERVAL=30
EOF

# 启动Agent
cd /opt/agent && ./agent start

echo "Agent installed successfully"
```

### 6.3 多服务器部署流程

```
1. 用户在Web界面点击"添加服务器"
2. 输入目标服务器IP + SSH端口 + 用户名 + 密码
3. 点击"测试连接"
   → SSH连接到目标服务器
   → 执行 `curl http://中心节点:5000/static/install-agent.sh | bash -s http://中心节点:5000 服务器ID`
   → Agent启动，开始采集数据
4. 目标服务器出现在服务器列表
5. 之后目标服务器的Agent每30秒自动上报数据
```

```
    },
```

```python
        "message": "服务器 {hostname} 磁盘使用率已达 {value}%"
    },
    "process_down": {
        "metric": "process_exists",
        "operator": "==",
        "threshold": False,
        "duration": 60,  # 1分钟

        "title": "进程异常退出",

    "server_offline": {
```

## 十三、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| SSH连接不稳定 | 添加服务器失败 | 添加重试机制；支持密钥认证 |
| Agent上报延迟 | 数据实时性差 | 优化采集频率；本地缓存 |
| 数据库膨胀 | 查询性能下降 | 定期归档历史数据；分表 |
| 容器内数据采集不到 | 监控盲区 | Agent部署在宿主机；共享docker.sock |
| 多服务器时间不同步 | 数据不一致 | 要求NTP时间同步；记录采集本地时间 |

# 权限系统清理分析

## 现状

两套权限系统并行运行，存在功能断裂：

### 系统A：角色管理（已废弃但代码和表还在）
- `Permission` 表（旧权限码，标注废弃）
- `CompanyRole`（角色定义）
- `CompanyRolePermission`（角色→旧权限关联）
- `UserCompanyRole`（用户角色分配）
- `UserCompanyPermission`（用户公司逐条权限）

### 系统B：权限矩阵（UMP位掩码，实际运行期使用）
- `Module`（通过 modules.py 自动注册）
- `ModuleAction`（模块动作定义，位掩码位置）
- `UserModulePermission`（用户×公司×模块，位掩码存储——一条记录=所有权限）

### 核心问题
角色分配写 `UserCompanyPermission`，但 `RoleRequired` 运行时校验读 `UserModulePermission`。
角色分配只控制了公司访问（用户能进哪个公司），功能权限一个都管不了。

### 用户结论
从用户使用角度：角色管理没起作用，权限矩阵已完全覆盖。
废弃角色管理，统一到 UMP 系统，可行。

- `core_user_company_role`（用户角色分配）

## 迁移策略

逐步修改，逐步验证，不一步到位：
1. 先查库确认数据状态
2. 挑一个点修改→验证
3. 逐步铺开→逐步验证
4. 全接管后再清旧表
